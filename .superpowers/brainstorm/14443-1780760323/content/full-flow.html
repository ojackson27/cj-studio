Section 3: The Complete Workflow

End-to-end — from first client contact to Vercel deploy. Human gates marked in purple, Claude execution in green.

0
Phase 0 · Human Client Discovery & Classification
→ Send Tally.so intake form (5 questions)
→ Read answers → classify into one of 4 tracks
→ Open the correct tracks/[type].md playbook
→ Fill in templates/brief.md with client answers
🔴 Gate: Edge cases (conflicting answers) → discovery call → you decide the track manually
1
Phase 1 · Human + AI assist Design Direction & Asset Gathering
You do:
→ Build brand palette (Coolors.co from photos)
→ Select typography pair (Fontshare)
→ Source/enhance photography (Remove.bg, Cloudinary)
→ Pull 5–10 reference screenshots (Mobbin / Awwwards)
→ Build brand kit in Canva Pro
Claude assists:
→ Generate moodboard description from references
→ Suggest page structure for the track
→ Draft copy for key sections from brief
→ Generate hero images via Midjourney prompt
🔴 Gate: Track-specific approval (photography / copy / visual direction) before Phase 2
2
Phase 2 · Claude Architecture & Brand Rules
Claude reads: tracks/[type].md + templates/brief.md → writes ARCHITECTURE.md with client hex codes, fonts, component list, and page structure locked.
3
Phase 3 · Claude Spec + Subagent Execution
Spec doc → implementation plan → subagent per component → spec compliance check after each. Prompt caching keeps token cost minimal across the full build.
🟢 Demo gate: Share Vercel preview URL with client after each major section. Adjust. Continue.
4
Phase 4 · Claude + Human review QA & Aesthetic Enforcement
/impeccable polish → strip template tropes → /impeccable document → lock to DESIGN.md. You review localhost render and apply final CSS blending.
5
Phase 5 · Human Client Approval & Vercel Deploy
Client signs off → final 50% invoice sent → domain pointed to Vercel → UptimeRobot monitor added → launch email + retainer link sent.

This is the full design. Happy with the complete workflow? Reply in the terminal.
